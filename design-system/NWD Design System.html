09SYSTEM · 09/13

Track lines

Linie tras niosą dwa wymiary jednocześnie: kolor = trudność (Green / Blue / Red / Black — standard MTB / FIS), animacja + glow = stan przejazdu (training / armed / verified / invalid). Nigdy nie mieszamy tych osi — kolor accent zarezerwowany dla stanu jest nakładany jako overlay (ring, glow, gate marks), nie jako stroke linii.

Difficulty taxonomy · MTB standard

GREEN S0 / S1
Łatwa. Szutry, bermy. Beginner.
BLUE S2
Średnia. Roots, bermy, drobne stopnie.
RED S3
Trudna. Steep, drops, kamienie.
BLACK S4
Expert. Big drops, steep tech, gap jumps.
2× BLACK S5
Pro / Elite. Wyłącznie ranked.
UNCAT —
Trasa user-generated, nie zwalidowana.
ZASADA #2 · DWIE OSIE

Stroke = trudność. Nigdy nie maluj armed-zielonym całej linii — to konflikt z difficulty. Stan przejazdu pokazujemy obok / dookoła linii: glow ring, animowany gate marker, status pill nad SVG. Gradient fill pod elevation profile zawsze dziedziczy kolor difficulty (z opacity 0.15 → 0).

Stroke biały (BLACK difficulty) + accent jako overlay armed: 1) blur-3px aura pod stroke, 2) 1px ring 16×16 dookoła gate squares 8×8, 3) splity i deltas pod osią X. Card border + soft glow w obrębie kontenera — nigdy nie wylewa się poza box.

3 line types · w kontekście difficulty

FLOW BLUE
Gładkie krzywe — bermy, jumpy. Cubic bezier, łagodne tranzycje.
TECH RED
Kanciaste segmenty — korzenie, kamienie. Linijki, miter joins, brak wygładzania.
GRAVITY BLACK
Drop heavy — DH, bike park. Mała progresja, ostry zjazd na końcu.

State coloring · ten sam Blue track w 4 stanach

TRENING
Difficulty stroke @ 55% + dashed. Gates muted.
ARMED
Stroke = difficulty. Aura blur + accent rings = state.
VERIFIED
Wszystkie gate rings persistent — przejazd zaliczony.
INVALID
Brakujący gate · X mark · post-fail line ghosted.
✕ ANTI-PATTERN
Nigdy nie zamieniaj stroke difficulty na accent kiedy linia jest armed. To zabija kontekst trasy ("jakie to było — Blue czy Red?") i myli z hierarchią state. Stroke = trudność na zawsze. State = warstwa overlay.

13REFERENCE · 13/13

Handoff dla Claude Code

Pakiet markdown + tokens.ts + theme.css w folderze design-system/. Kolejność implementacji:

# 1. Drop theme.css w global stylesheet (CSS vars + keyframes)
import './design-system/theme.css';

# 2. Import tokens.ts jako single source. Nigdy nie hardcoduj koloru.
import { tokens } from './design-system/tokens';
// ✓ tokens.palette.acid.accent
// ✗ '#00FF87'

# 3. Build base components per components.md:
#    Btn, Pill, HudPanel, RaceTime, LeaderboardRow, IconGlyph

# 4. Apply patterns.md do istniejących ekranów:
#    - Run flow narrative (6 ekranów × stan kolor)
#    - Race-state pattern wszędzie gdzie jest run summary
#    - Empty states: brak ilustracji, krótki copy + CTA

# 5. Audyt copy względem voice.md, podmień ad-hoc icons
#    na 12-glyph set z icons.md
NON-NEGOTIABLES
  • Numbers are tabular-nums — zawsze. Race times nie skaczą.
  • Race state owns color — nie używamy accent do dekoracji.
  • Reduced motion is honored — wszystkie animacje gated.
  • No light mode — produkt to night-of-race energy.
  • No emoji in UI — tylko IconGlyph.
  • CTA = CAPS + imperative — nigdy "Click here".
